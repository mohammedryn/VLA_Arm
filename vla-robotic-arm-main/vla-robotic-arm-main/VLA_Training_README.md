# VLA Robotic Arm — Training Guide

## Overview

This repository contains a full Vision-Language-Action (VLA) training pipeline
for a 4-DOF robotic arm controlled by a Raspberry Pi 5 and Teensy 4.1.

---

## Dataset Pipeline

| File | Role |
|---|---|
| `dataset/hdf5_reader.py` | Load HDF5 demos; convert raw telemetry bytes to structured arrays |
| `dataset/skill_segmenter.py` | Rule-based skill labeller (REACH → GRASP → LIFT → PLACE) |
| `dataset/augmentation.py` | 4-way data augmentation (identity, joint noise, load noise, flip) |
| `dataset/vla_dataset.py` | `torch.utils.data.Dataset` wrapping the full pipeline |

---

## Colab Training Notebook

The Colab notebook cells are documented **in order** — run them top-to-bottom
without skipping. Each cell header states its purpose and any required
prerequisites (GPU runtime, mounted Drive, etc.).

Approximate cell order:

1. **Environment setup** — install dependencies, mount Google Drive
2. **Dataset path** — point `DEMOS_DIR` at your `demos/` folder in Drive
3. **Load & segment** — `load_all_demos` + `segment_demo` for every HDF5 file
4. **Augment** — `build_training_set(all_timesteps, augmentation_factor=4)`
5. **Dataset & DataLoader** — instantiate `VLADataset`, configure `DataLoader`
6. **Model definition** — VLA policy head (ResNet vision + T5 language + MLP action)
7. **Training loop** — loss, optimizer, gradient clipping, checkpoint saving
8. **Evaluation** — validation loss curve and per-skill accuracy

## Starting Real Training

Real training begins once the `demos/` folder is populated with HDF5 files
recorded from teleoperation. Each file must follow the layout written by the
RPi5 data-collection script:

```
demos/
  task_001.h5
  task_002.h5
  ...
```

HDF5 layout per file:

| Dataset | dtype | shape | notes |
|---|---|---|---|
| `/telemetry` | `uint8` | `(T, 250)` | raw `TELEMETRY_DTYPE` bytes at 50 Hz |
| `/rgb_frames` | `uint8` | `(F, H, W, 3)` | gzip-compressed RGB frames |
| `/frame_ts` | `uint64` | `(F,)` | frame timestamps in microseconds |

Required HDF5 attributes: `instruction` (str), `task_type` (str).

Until `demos/` contains real HDF5 files, the pipeline runs on synthetic data
generated by `dataset/hdf5_reader.generate_synthetic_demo()`.
